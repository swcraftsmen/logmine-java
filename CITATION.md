# How to Cite

## Citing the Algorithm

If you use LogMine in research or production, please cite the original algorithm paper:

```bibtex
@inproceedings{hamooni2016logmine,
  title={LogMine: Fast Pattern Recognition for Log Analytics},
  author={Hamooni, Hossein and Debnath, Biplob and Xu, Jianwu and Zhang, Hui and Jiang, Guofei and Mueen, Abdullah},
  booktitle={Proceedings of the 25th ACM International Conference on Information and Knowledge Management (CIKM)},
  pages={1573--1582},
  year={2016},
  organization={ACM},
  doi={10.1145/2983323.2983358}
}
```

**Paper:** [https://www.cs.unm.edu/~mueen/Papers/LogMine.pdf](https://www.cs.unm.edu/~mueen/Papers/LogMine.pdf)

## Citing This Implementation

If you want to reference this specific Java implementation:

```bibtex
@software{logmine_java,
  title={LogMine: Java Implementation},
  author={[Your Name]},
  year={2025},
  url={https://github.com/[your-org]/logmine}
}
```

## In Text

When mentioning LogMine in papers or documentation:

> We used LogMine [1], a fast pattern recognition algorithm for log analytics, 
> implemented in Java [2].
>
> [1] Hamooni, H., et al. (2016). LogMine: Fast Pattern Recognition for Log Analytics. CIKM 2016.
> [2] [Your implementation reference]

## For Non-Academic Use

If you're using LogMine in production and want to give credit in documentation:

```markdown
This system uses LogMine, based on the algorithm by Hamooni et al. (CIKM 2016).
Implementation: https://github.com/[your-org]/logmine
```

## Why Cite?

- **Academic integrity** - Proper attribution to original researchers
- **Reproducibility** - Others can verify and reproduce your results
- **Credit** - Researchers get recognition for their work
- **Trust** - Shows your work is based on peer-reviewed research
